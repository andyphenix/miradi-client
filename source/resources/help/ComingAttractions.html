Future Improvements to Miradi Desktop Software

With the release of Version 4.0, Miradi desktop software has largely completed its initial development phase. That said, we will be continuously working to develop and improve many aspects of the software and issuing new releases on a periodic basis. A few highlights that we are working on include:

  • Upgrade of the Work Plan View - Major enhancements to the way in which data is entered and reported on in this view.
  • Enhanced Import and Export - Improved ability to export information in various formats including to automatically generate documents that contain the core of material needed for reports to key conservation donors and organizations.
  • Tools to Support Later Steps in the CMP Open Standards - The current version of Miradi is largely focused on the planning phase of the adaptive management cycle; future tools will help in the analysis and learning phases.
  • Translations to Additional Languages - Miradi is now available in a variety of different languages such as Spanish, Chinese, and Bahasa Indonesia. See the download page on the Miradi.org website for more details. If you are interested in helping us develop a version for your language, please contact us.
  • Adaptations to Other Disciplines - Miradi currently focuses on biodiversity conservation projects. However, the adaptive management approach underlying Miradi undoubtedly will be of use to other disciplines as well, such as development projects, public health projects, or many others. If you are interested in helping us to develop a version for your field, please contact us.

If you have additional ideas or requests, please contact us at: feedback@miradi.org.  And please check the Miradi website (www.Miradi.org) for the latest version of the software.

Introducing Miradi Share: Bringing Miradi Data Online to Manage Programs

Although Miradi Desktop software provides a powerful tool to manage individual conservation projects, there is a growing need to aggregate and combine data from individual projects into the broader data sets needed to design, manage, monitor, and learn from collections of projects (aka "programs"). To this end, Miradi Share is a new web-based system that enables Miradi users to upload their projects to an online database. Miradi Share enables project teams to:

  • Enter Data in a Consistent Fashion - Coordinate the way data is entered into multiple Miradi files within a program.
  • Search For, Download & Learn from Similar Projects - Find other projects that have done similar work so that you do not have to "reinvent the wheel."
  • Work Collaboratively on Projects - Have multiple team members work on and update a Miradi project file in a coordinated fashion.
  • Undertake GIS and Other Online Analyses - Use online tools to extend currently available desktop analyses. (For example, creating a Miradi-linked GIS platform to help make the Open Standards more spatially explicit.)

Moving beyond individual "child" projects, Miradi Share also enables "parent" program managers to:

  • Aggregate Project Information into Program Plans - Create higher level program plans that roll-up information from their component projects.
  • Track Project & Program Performance - Monitor and manage project and program implementation.
  • Generate Reports Needed to Manage the Program - Obtain and analyze data needed to create information for management and performance reporting purposes.
