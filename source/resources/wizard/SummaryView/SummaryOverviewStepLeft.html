Welcome to the Miradi Interview & the Summary View!!

As you go through the steps in the CMP Open Standards, this leftside panel of the interview will provide you with step-by-step instructions. The rightside panel will provide definitions and guidance.

The <Project> tab below provides an overview of your project. You can fill in your Project Name and set the Primary Project Data Language and the Project Data Effective Date (the date of your most recent substantive updating of the Miradi project file).

A new feature of Miradi 4.1 is that is now compatible with Miradi Share! Click on the "Learn about Miradi Share" link or go to www.miradishare.org to find out more about this new online tool that greatly extends the capability of Miradi Software.

Click next to define your initial project team.

Later on, you can return to this page and fill in the remaining fields.The Project Number field can be used for any numbering system your organization uses. Related Projects can be used to describe other sibling or parent projects. You can put your project's website adress in the Project Website field. The Project Description field can be used for a summary or abstract of your project. The Project Status field can be used to generally describe how your project is doing at the moment. The Next Steps field can be used to describe any critical next steps; this field is especially useful when you want to send your project file to other team members. Finally, the Lessons Learned field can be used to descibe any key learnings.

Hint: Note that Project Filename is read only as shown by the grey background. To change what appears in this field, you have to rename the project file itself. This can be done from the initial Welcome (No-Project) page.

Hint: The External Project IDs field is another read only field used by external databases such as Miradi Share.
