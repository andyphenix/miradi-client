Welcome to the Miradi Interview & the Summary View!

As you go through the steps in the CMP Open Standards, this leftside panel of the interview will provide you with step-by-step instructions. The rightside panel will provide definitions and guidance.

The <Project> tab below provides an overview of your project. You can fill in your Project Name and set the Primary Project Data Language and the Project Data Effective Date (the date of your most recent substantive updating of the Miradi project file).

Miradi is now enhanced by Miradi Share. Click on the "Learn about Miradi Share" link or go to www.miradishare.org to find out more about this online tool that greatly extends the capability of Miradi Software.

Click the <Next> button to define your initial project team.

Later on, you can return to this page and fill in the remaining fields:

  • Project Number field can be used for any numbering system your organization uses.
  • Related Projects can be used to describe other sibling or parent projects.
  • Project Website is where you can put your project's website address.
  • Project Description can be used for a summary or abstract of your project.
  • Program Classifications are used with Miradi Share to tag your project with key info.

The Progress Report fields can be used to update the progress of your project on a certain date. Click the <Progress Report> side tab, and then click the <Create Progress Report> button to create a progress report with the following fields:

  • Project Status can be used to generally describe how your project is doing at the moment.
  • Next Steps can be used to describe any critical next steps; this field is especially useful when you want to send your project file to other team members.
  • Finally, the Lessons Learned field can be used to describe any key learnings.
Miradi will display the most recent progress report data on the main Summary side tab.

Hint: Note that Project Filename is read only as shown by the grey background. To change what appears in this field, you have to rename the project file itself. This can be done from the initial Welcome (No-Project) page.

Hint: The External Project IDs field is another read only field used by external databases such as Miradi Share.
