Navigation Hints for the Summary View and the Overall Application

Summary View
The first few tabs in the Summary View contain basic information fields that most projects will want to fill in about their project, team, scope, location, and planning. This basic information will be especially important when you export your project to various reports and databases. Some organizations also have their own custom data fields for their projects. You can find these by clicking on the appropriate tab.

Overall Application

All views have an interview pane on top and an action pane below. Use the <Previous> and <Next> buttons to move backwards and forward through the interview. You can resize or hide the interview pane by using the splitter and/or arrows. In many dialog boxes, you can use the <Instructions> button at the bottom left to go the appropriate interview page. You can also often use the right (alt) mouse button to get context sensitive menu commands (Mac users with one button mice can hold the Ctrl key while clicking). Use the buttons on the upper right to see a dashboard overview of your project or to get context sensitive additional information, examples, and workshop hints.

Note that the program automatically saves all changes so there is no "save" command. You can save a copy under a different name using the <File/Save As> menu command. Click on the <File/How To Save> menu command to get more details about saving files in Miradi.

Data fields with blue text on a white background have user-entered and editable data. Data fields with black text on a grey background have read-only data that has been entered (and generally can be edited) elsewhere in the application. You can also now right click (Ctrl-click in Mac) in any multi-line data field to get basic text formatting commands such as bold, italic, or to insert a hyperlink.
