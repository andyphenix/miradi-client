You have constructed your conceptual model in a series of separate steps. At this point, you should take a look at your model as a whole and ask yourself the following questions:
  • Does this model reasonably portray what is happening at my site?
  • Are all the key targets represented?
  • Are all the key direct threats represented?
  • Are all the key contributing factors (indirect threats and opportunities) represented?
  • Are the link arrows properly placed?
  • Have you verified that nothing major is missing?
  • Will the model be understandable to my project team?
  • Will the model be understandable to people outside of my project team?

If you answered "yes" to all of these questions, click the <Next> button below to continue. If you answered "no" then you need to go back and fix your model.

Hint: Tags can be a powerful tool to help manage the visual complexity of a model.

New Feature Hint: Tags have been updated; here are the major changes:

  1. A checkbox has been added next to the <Manage Tags> button on the toolbar to “enable” the application of the tags. When this box is unchecked, all factors are displayed in the diagram. When this box is checked, only factors with the checked tags are displayed.
  2. The application of a tag to a factor is now diagram-specific. In the past, when a tag was applied to a target, it applied for every conceptual model and result chain that contained that target. Now, checking a tag for a factor applies that tag only for the active diagram. In migration from prior Miradi versions, all prior tag applications are retained, so conscientious taggers will likely want to cleanup their prior tag applications.
  3. Only the tags that have been applied to factors in the active diagram are displayed in the list in the Control Bar. In the past, all tags were displayed in every diagram.
  4. All menu options to add and remove tags have been modified to be diagram-specific. A “Remove All Tags” option has been added to the right-click context menu; this option clears all tags applied to the active diagram for one or more selected factors. The “Remove Tag” option displays only the tags that have been applied for the selected factors for the active diagram, and clears the selected tag for the active diagram only. The Add Tag options display all tags so that tags may be applied to diagrams for the first time.

To employ tags:

  1. Click on the <Manage Tags> button on the toolbar.
  2. Create one or more tags. They can be either related to a theme (e.g. all factors that are part of a specific chain), or a sequence in which you want the tagged items to appear (e.g. in an "animated" powerpoint). If you want to specify a sequence, it is also helpful to put numbers or letters in order in the ID field.
  3. Returning to the diagram, hold the shift key down and select the factors that you want to apply a tag to, and then use the <Actions/Tag/Untag...> menu command, or the <Add Tag to Selected Items> command on the right (alt) click mouse button to apply the tag.
  4. Repeat with other tags as needed.
  5. You can then use the check boxes in the control bar to set the visibility of each tag group.
  6. Remember that once at least one tag is checked, then only checked tags will be visible.
