Exporting Workshop Data in Miradi to Other Applications

Although you may not want to use Miradi to display information in a workshop, it is a very useful way to capture the information put on a sticky board or a flip chart so that it can be documented and shared with other workshop participants. There are many ways to export your information from Miradi to other applications.

Export Images

Use the <File/Export/JPEG Image> or <File/Export/PNG Image> menu commands to capture a specific diagram or table on your screen in an image file. This method is particularly useful to:
  • Put a diagram or table in a document or presentation
  • Create an image that can be put in a website

Export Miradi Zip File

Use the <File/Export/Miradi Project Zipfile> menu command to capture your entire project in a small zip file. This method is particularly useful to:
  • Send your project to another Miradi user
  • Create a backup copy of your project

Export Project XML File

Use the <File/Export/Project XML File> menu command to create an XML file with all your project data. Although managing the XML file will require technical skills, this method is particularly useful to:
  • Create a file that can be imported by a database
  • Export all of your project's information into formats that can be read by other applications

Export Tab Delimited Data

Use the <File/Export/Tab Delimited Data> menu command to export the currently viewed table to a tab-delimited file. This method is particularly useful to:
  • Export the current configuration from your planning view to a spreadsheet like Excel
  • Export financial information

Generate Reports

Use the <Reports> view to create a PDF (Adobe Acrobat) or ODT (Open Office) file that contains various configurations of your project data. This method is particularly useful to:
  • Create a document that can be edited in the Open Office Word Processor
  • Create a standard report for a donor or other key audience for your project
