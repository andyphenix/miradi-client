Exporting and Importing from Miradi - Preview of Coming Attractions

Miradi is stand-alone desktop software.  But we recognize that project teams will need to both import data from and export data to many different sources.  Some of the key functions we hope to create include:

Configure Export

This command will enable you to define what parts of your project file you want to share with other people.  For example, when uploading your project to a public database, you may wish to not upload financial data or information about the locations of sensitive species.

We hope to make Miradi "fully granular" in configuring what to export.  We also will provide a way to encrypt or otherwise protect sensitive information.  Benetech has lots of experience with this sort of requirement from its work with the Martus Human Rights software.

Export

This command will enable you to export your data to many different sources.  For example, you might wish to export:

  • Financial Data - You might want send your budget or actual expenditures on a quarterly basis to your organization's accounting system, thus keeping the bean counters happy. 
  • Project Summary Data - You might also want to regularly upload your project's summary data to your organization's database of projects.  Imagine the convenience of not having to "double-enter" information into a website, but instead having it come straight from the software you are already using to manage your project.
  • Donor Applications and Reports - Most conservation projects have to write half-a-dozen or more funding proposals each year, and even more reports on the grants that they have.  Typically these applications and reports require similar data (project goals, objectives, monitoring results), but in different formats.  If we can get pre-load the templates for the applications and reports for major donors into Miradi, it should be possible to then export your project information into a word processor file that can then be lightly edited and sent to the donors.  And just maybe someday, we can even convince some donors to directly accept Miradi Project files as reports themselves! 
  • Project Results and Lessons Learned - There are many project teams around the world who undoubtedly could benefit from your experiences and the lessons you have learned.  We hope to create a series of linked databases that would then house information contributed by projects from around the world.  These databases would be the ultimate foundation for cross-project learning.

Import

This command will enable you to import data from many different sources.  For example, you might wish to import:

  • Basic Project Information - Information from another program.
  • Financial Data - Budget information from an Excel spreadsheet.
  • Maps - Base geographical maps created in a GIS program.
